解析Java框架中entity层，mapper层，service层，controller各层作用

发表于 2019-11-05
字数统计 730 | 阅读时长 3

转：CSDN，个人感觉讲的比较清晰。

一、entity层

别名： model层 ，domain层
用途： 实体层，用于存放我们的实体类，与数据库中的属性值基本保持一致，实现set和get的方法。
例子：user表的实体User

二、mapper层

别名： dao层
用途： 现在用mybatis逆向工程生成的mapper层，其实就是dao层。对数据库进行数据持久化操作，(数据持久化操作就是指，把数据放到持久化的介质中，同时提供增删改查操作，比如数据通过hibernate插入到数据库中)，他的方法语句是直接针对数据库操作的，主要实现一些增删改查操作，在mybatis中方法主要与与xxx.xml内相互一一映射。
示例：userMapper

三、service层

用途：业务service层， 存放业务逻辑处理，给controller层的类提供接口进行调用。一般就是自己写的方法封装起来，就是声明一下，具体实现在serviceImpl中，在接口的实现方法中需要导入mapper层 。
示例：UserService

四、controller层

别名：web 层
用途： 控制层，负责具体模块的业务流程控制，需要调用service逻辑设计层的接口来控制业务流程。因为service中的方法是我们使用到的，controller通过接收前端H5或者App传过来的参数进行业务操作，再将处理结果返回到前端。
示例：UserController
